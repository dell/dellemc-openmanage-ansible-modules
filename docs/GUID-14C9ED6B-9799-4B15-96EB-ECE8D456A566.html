Key Features

The key features that are supported in OpenManage Ansible Modules are:

  • Quick Deploy settings of the server or IOM on OpenManage Enterprise Modular.
  • Log in security properties settings on OpenManage Enterprise and OpenManage Enterprise Modular.
  • Session inactivity timeout settings on OpenManage Enterprise and OpenManage Enterprise Modular.
  • Local access settings on OpenManage Enterprise Modular.
  • SMTP or email configurations on OpenManage Enterprise and OpenManage Enterprise Modular.
  • Syslog forwarding settings on OpenManage Enterprise and OpenManage Enterprise Modular.
  • Local access settings on OpenManage Enterprise Modular.
  • Chassis network services settings on OpenManage Enterprise Modular.
  • Device location settings on OpenManage Enterprise Modular.
  • Network settings of devices on OpenManage Enterprise Modular.
  • Chassis power settings on OpenManage Enterprise Modular.
  • Manage OpenManage Enterprise profiles.
  • Configure compliance configuration baselines and generate compliance report on OpenManage Enterprise.
  • Discover devices, create static group, and add devices to a static group on OpenManage Enterprise.
  • Export SupportAssist device collections from OpenManage Enterprise and OpenManage Enterprise Modular.
  • Configure Active Directory service connection and manage Active Directory user group on OpenManage Enterprise.
  • Redfish subscriptions on iDRAC.
  • Update firmware and manage storage volume configuration using standard Redfish URIs supported by iDRAC.
  • Manage network, proxy, web server, and time configurations.
  • Create, modify, or delete single identity pool.
  • Create a firmware baseline or catalog.
  • Retrieve baseline and baseline compliance details.
  • Create, modify, or delete user accounts.
  • Power state management operations on devices managed by OpenManage Enterprise.
  • Create, modify, deploy, export, import, and clone a template.
  • Get the list and details of all user accounts or of a specific account.
  • Get the list and details of templates or of a specific template.
  • Firmware update of PowerEdge devices and all its components.
  • Retrieve job details for a given job ID or the entire job queue.
  • Retrieve the list of all devices with the exhaustive inventory of each device.
  • Export a server configuration profile (SCP) containing either the entire server configuration or component level configuration (iDRAC, BIOS, RAID, NIC) to a local file path on Ansible controller or a remote network share.
  • Import an SCP from a local file path on Ansible controller or a remote network share.
  • BIOS, Integrated Dell Remote Access Controller (iDRAC), NIC, and RAID.
  • Firmware update using a firmware repository that is hosted on CIFS, NFS, HTTP, HTTPS, or FTP shares.
  • View firmware inventory details.
  • Support for Windows, Linux, and ESXi operating system deployments.
  • Configure power controls, resetting iDRAC, viewing Lifecycle Controller (LC) job status, deleting LC job, deleting LC job queue, exporting LC logs, and configuring system lockdown mode.
  • Retrieve the system inventory details.
NOTE: These features are supported only on iDRAC with enterprise license.
